Reinforcement Learning

Reinforcement Learning Machine Learning Decision Theory
TL;DR

Reinforcement learning (RL) is how a computer learns to make decisions by trying them out. An agent takes actions in an environment, receives rewards, and gradually improves the rule it uses to pick actions (its policy). Unlike supervised learning, nobody tells it the right answer — only whether things went well.

Overview

Think about how a dog learns a trick. You don't hand it a labeled dataset. You let it try things — and when it does something you like, it gets a treat. Over time, it learns which actions tend to earn treats in which situations. That is essentially the RL setup.

Five ingredients show up in almost every RL paper:

Agent
The learner / decision-maker. This is what we're training.
Environment
Everything outside the agent — the world it acts in.
State (s)
A snapshot of the environment the agent can observe.
Action (a)
A move the agent can make.
Reward (r)
A scalar number the environment returns after each action. Positive = good, negative = bad.

The agent's goal is simple to state and hard to achieve: maximize the total reward it expects to collect over time.

The Core Loop (Visual)

Policy, Value, and the Tradeoff

Two quantities do most of the work in RL:

Policy (π)
The agent's strategy — a rule that maps states to actions (or to probabilities over actions). "In this situation, do this."
Value function (V or Q)
An estimate of how good it is to be in a state (or to take a specific action in a state), measured by the total future reward the agent expects to get.

A big part of RL theory is figuring out the relationship between policies and values — and how to update one using the other.

Exploration vs. exploitation. Early on, the agent has no idea which actions are good, so it has to explore. But if it explores forever, it never profits from what it's learned. If it exploits too early, it locks in on a mediocre strategy. Every RL algorithm has to manage this tradeoff — usually with a schedule (e.g. "random 10% of the time") or a smarter scheme.

Markov Decision Processes (the formal stage)

Most RL assumes the environment is a Markov Decision Process (MDP): a setting where the next state depends only on the current state and action, not on the entire history. An MDP is described by four things:

MDP = (S, A, P, R)
S = set of states · A = set of actions
P(s' | s, a) = transition probability · R(s, a) = reward function

Once you cast a problem as an MDP, the goal is to find a policy π that maximizes expected cumulative reward — often written:

maxπ E[ Σt γt · rt ]

The γ (gamma) is a discount factor between 0 and 1 — it controls how much the agent cares about long-term vs. short-term rewards.

The Main Families of RL Algorithms

Value-based

Learn Q(s, a) — the expected value of taking action a in state s. Act greedily with respect to it. Examples: Q-learning, DQN.

Policy-based

Directly parameterize and optimize the policy itself. No value function needed. Examples: REINFORCE, PPO.

Actor–Critic

Mixes both: an "actor" picks actions (policy) and a "critic" estimates values. Most modern deep-RL algorithms live here.

Model-based

Learn a model of the environment's dynamics, then plan inside it. Data-efficient but harder to get right. Examples: Dyna, MuZero.

Real-World Examples

Game playing

AlphaGo, AlphaZero, and OpenAI Five all learned superhuman play through RL — state = board/game, action = move, reward = win/loss.

Robotics

Teaching a robot arm to grasp or a quadruped to walk. Actions are motor torques; rewards encode stability, speed, or task progress.

RLHF in LLMs

Reinforcement Learning from Human Feedback — the technique behind ChatGPT-style training. Human preferences become the reward signal.

Recommendation systems

Framed as choosing what to show a user to maximize long-term engagement, not just the next click.

Operations research

Inventory management, energy grid control, traffic light scheduling — anything with sequential decisions and delayed payoff.

Drug & molecule design

Treat molecule construction as a sequence of "add this atom" actions, with reward tied to desired chemical properties.

Why RL Is Hard (and Interesting)

  • Credit assignment. If you win a chess game after 40 moves, which moves were actually responsible? Reward may be delayed by many steps.
  • Sample efficiency. Deep RL typically needs millions of interactions. Fine for simulators; painful in physical systems.
  • Reward design. Poorly-specified rewards lead to "reward hacking" — agents exploit loopholes rather than solve the intended task.
  • Stability. Neural-network-based value functions can diverge during training. Lots of modern research is about taming this.
